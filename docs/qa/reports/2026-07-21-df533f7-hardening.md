## Implemented in this change set

| Area | Status | Evidence |
| --- | --- | --- |
| Roundtable participant policy | Implemented | API validation and database wrapper require exactly three distinct instances and at least two provider families. |
| Sporttery offer boundary | Implemented | Database eligibility predicate requires an authorized source, eligible competition, verified fact, `on_sale`, pre-cutoff state, matching source provenance, and an odds snapshot. |
| Empty autonomous selection | Implemented | The database returns `no_eligible_sporttery_matches`; it does not create an empty job. |
| Lifecycle bootstrap | Implemented through notarization | Lifecycle delivery calls an idempotent database initializer; the real local run completed `predict_score` → debate → vote → bet → notarize. |
| Worker executor configuration | Implemented as production factory | `ALEA_PHASE_EXECUTOR_FACTORY=app.workers.production_executor:create_phase_executor`; missing or invalid factories fail closed. |
| Unlicensed history ingestion | Blocked by policy | The backfill orchestrator requires all five authorization capabilities and contains no web scraper. |
| Supabase/JWT fallback | Removed | API and middleware no longer fall back to a hard-coded remote project URL; missing issuer configuration fails closed. |
| Consent versioning | Externalized | Signup refuses to create an account until active terms, privacy, and risk versions are configured. |
| Liveness/readiness | Separated | `/health` and `/livez` are process liveness; `/readyz` checks database, migration, Redis, Supabase, Dispatcher, Worker and executor factory. |
| CI and repository hygiene | Added | GitHub Actions, generated-artifact checks, conservative cleanup report, and ignore rules are included. |
