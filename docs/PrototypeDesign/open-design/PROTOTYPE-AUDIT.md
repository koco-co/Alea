# Alea 原型静态验收报告

> 复核日期：2026-07-18
> 唯一需求基线：`PRD.md` v1.4
> 正式游客入口：`alea.html`
> 验收性质：代码与产物静态验收，不等同于真实服务、OAuth 或浏览器完整 E2E

## 1. 结论

PRD §18 声明的 12 项 P0 原型交付均已有对应页面、状态与跨页映射；P1 第 13 项「排行 + AI 实例主页」、第 14 项「盈亏页」与第 15 项「复盘列表 + 复盘详情」也已在正式控制台内完成。集中静态回归结果为：

- 核心静态契约：`PASS 167` / `FAIL 0`
- 弹层与模态焦点契约：`PASS 6` / `FAIL 0`
- 排行与 AI 实例主页契约：`PASS 20` / `FAIL 0`
- 盈亏页契约：`PASS 22` / `FAIL 0`
- 复盘列表与详情契约：`PASS 26` / `FAIL 0`
- 7 个 HTML 文件均通过脚本语法、重复 `id`、重复 `data-od-id`、CSS 变量与本地引用检查。
- 官方 Open Design 导出成功，`alea-console.html` 的 AI 实例详情、盈亏页与复盘详情均未发现应用层重叠、文字裁切或图表溢出；长图分段拼接会重复捕获粘性头部，不属于 DOM 重复。

上述结果只证明本报告列出的静态范围。剪贴板写入、真实 OAuth、后端鉴权、网络同步、逐像素多视口视觉与动效真实时序仍需运行态测试。

## 2. 旧报告中的误报与口径错误

旧版审计不再作为当前状态依据，主要原因如下：

1. `assets/deepseek.svg`、`assets/anthropic.svg`、`assets/gemini.svg`、`assets/alibaba-cloud.svg`、4 个球队队徽 PNG 与 `assets/hero-wc-final.jpg` 实际均存在；“9 个资源缺失”是误报。
2. AI 阵容、数据同步、系统设置在 PRD §18 的编号 15–17 中均标为 P1，不是 P0；旧报告把它们升级为 P0，违背 PRD 优先级。
3. 比赛详情的赛果与复盘 Tab 在 PRD §18 编号 4 中明确标为 P1；P0 只要求竞彩、情报、预测三个优先 Tab。
4. 控制台未定义组件类、原生 `confirm()`、固定 SVG 高度、旧演示数据和跨页采纳参数是实际缺陷，已按 PRD 修复。

`PRD.md` 本轮保持不变：它继续作为唯一功能与优先级基线；被纠正的是审计报告对 PRD 的错误解读。

## 4. 本轮修复清单

### 首页与品牌

- 无 JavaScript 时 Hero 标题、说明、行动按钮和证明数据保持可见。
- Canvas 在容器宽高为零时直接降级，不再进入无效绘制。
- 圆桌演示只播放一次并收敛；环境粒子在 4.8 秒停止，保留 `prefers-reduced-motion`。
- 清除世界杯、西班牙、阿根廷、梅西及旧控制台球队数据，统一为澜城竞技、赤湾联、北境工业与海岬城。
- 六个核心 OKLch 令牌与 `brand-spec.md` 一致；HTML 样式不再引入 raw hex 或 `rgba()`。

### 认证与控制台

- 认证 Tab 改为真正的 `button[role=tab]`，补全方向键切换与焦点状态。
- 移除默认预填密码；演示账户改为用户主动触发的填充操作。
- OAuth 注册在 18 岁和条款确认前不可用；登录成功保留安全回跳。
- 补齐控制台实际使用的 banner、状态、表单、Tab、评分卡和选项网格样式。
- 未知日历日期进入真实空态，不再错误回退到“明天”。
- 控制台通知图标改为可访问 SVG；旧 `adopted=1` 改为公证 `N8C4-02`。
- 控制台、预测页与管理端头部菜单按非模态弹层实现，打开后聚焦首个可操作控件，Escape 关闭并返回触发按钮。

### 预测、算票与管理端

- 在售单场与串关使用公证编号跨页采纳；计算器按 `N8C4-02` / `P3L2-04` 精确还原配置、串关、2 倍与金额。
- 部分结算串关保留四腿状态和退款计算，但不再提供错误的采纳操作。
- 竞彩规则迁入带版本号的 `application/json` 数据块，逻辑只读取该配置。
- 原生 `confirm()` 全部替换为品牌确认面板，并提供 Escape、焦点返回与预算二次确认。
- 计算器和管理端真正的模态操作框限制 Tab 焦点在框内；管理端确认框使用视口居中遮罩层，取消或 Escape 后回到原触发控件。
- 删除重复的旧 SVG 出图函数；导出高度按腿数计算，Canvas 使用同一动态高度。
- 方案卡生成时间按 `Asia/Shanghai` 实时生成；未采纳时标注“自主配置”，采纳时显示真实公证编号。
- 管理端补齐完整导航、通知与账户菜单；发布后写入同一公证状态并可直达用户预测。

### 排行与 AI 实例主页（P1 第 13 项）

- 排行主表补齐综合分、比分、胜平负、总进球、半全场与模拟盘收益率六维切换，以及全部时间/近 30 天/近 7 天和赛事筛选。
- 三个 AI 实例均可通过独立 hash 路由进入同一正式控制台内的实例主页；返回后保持排行导航语义。
- 排行行展示近 10 场命中色条、模拟盘迷你净值曲线、周排名趋势与样本不足不授名次状态。
- 实例主页补齐身份、模型版本/参赛日期待接入口径、六维分值、校准分桶、完整净值图、预测历史、辩论回放入口与教训注入状态。
- 原型统计明确标注“情景演示 / 非真实战绩”；没有来源的数据统一显示“待同步”，不冒充真实公证账本数据。

### 盈亏页（P1 第 14 项）

- 圆桌共识账户置顶，明确初始资金、终投胜出方案、公证账本、申报/实际执行仓位，以及单日 15% / 同场 5% 风控口径。
- 主图支持 7 天、30 天与全部范围切换；圆桌共识、Claude-1、GPT-1、Qwen-1 四条曲线可独立显隐。
- 曲线区补齐相对净值坐标、指针十字线、当前结算点与公证账本日明细浮层规格。
- 图下汇总表覆盖账户、当前净值、总收益率、最大回撤、投入场次与命中场次六列；无来源的值统一显示“待同步”。
- `?pnl=empty` 进入上线初期空状态；真实盘未在用户端暴露，继续保留为 PRD P2 管理员能力。

### 复盘列表与详情（P1 第 15 项）

- 复盘列表改为卡片流，覆盖日期、赛事、参与 AI、命中与否四类筛选，并提供真实空状态。
- 命中与未中均可进入复盘；每张卡展示预测/实际比分、原因摘要、参与 AI 头像组与独立详情路由。
- 详情按 PRD 固定为赛况对照、逐 AI 检讨、共性失误、教训清单和关联证据五段。
- 逐 AI 检讨使用可键盘展开的原生 `details`；每个实例均展示当时判断、偏差与根因。
- 教训覆盖新建、已注入下次预测上下文、已归档三态；归档记录同时显示归档原因与操作日志。
- 本项只实现 PRD §12.1–12.2 的用户侧列表与详情；§12.3 管理端自动生成、审核和发布流程未越级并入。

## 5. 静态验收覆盖

本轮集中检查覆盖：

- 数据映射：管理端发布 → 用户预测 → 单场/串关采纳 → 算票器预填。
- 盈亏口径：共识账户与实例账户分离、范围切换、曲线显隐、指针明细、汇总表与空状态。
- 复盘口径：列表筛选、命中/未中双分支、五段详情、AI 折叠检讨、教训生命周期与证据回链。
- 动效触发：一次性圆桌、4.8 秒上限、减少动态效果、IntersectionObserver 降级。
- 会话过场：首页加载、认证回跳、管理端发布状态的 `sessionStorage` 传递。
- Canvas 降级：首页零尺寸保护；算票 PNG 动态高度。
- 旧演示数据：世界杯数据与旧控制台球队数据均无残留。
- 代码完整性：内联脚本语法、本地资源、CSS 变量、重复 `id`、重复 `data-od-id`。
- 无障碍基础：跳转链接、按钮 Tab、非模态/模态语义、打开聚焦、Tab 焦点循环、Escape 返回焦点、状态文字不只依赖颜色。
